# Audit Scope – AWS Cloud IT Audit

## 1. Audit Objective
The objective of this audit is to assess whether key AWS cloud controls are properly designed and implemented to mitigate security and operational risks.

## 2. In-Scope Areas
The audit will focus on the following control domains:

### Identity and Access Management (IAM)
- User and role management
- Privileged access
- Multi-Factor Authentication (MFA)
- Least privilege access

### Logging and Monitoring
- CloudTrail configuration
- Logging coverage across services
- Log integrity and retention

### Storage Security
- S3 bucket access controls
- Public access configuration
- Data protection risks

## 3. Out of Scope
- Network-level controls (VPC, firewalls)
- Application-level vulnerabilities
- Advanced threat detection tools

## 4. Audit Approach
The audit will be conducted through:
- Configuration review
- Evidence collection (screenshots, logs)
- Control testing
- Risk identification and rating

## 5. Expected Output
- Control test results
- Risk register
- Audit findings
- Recommendations for remediation
